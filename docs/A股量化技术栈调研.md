# A股量化交易完整技术栈调研

## 数据源对比

| 数据源 | 费用 | 适合场景 | 注意 |
|--------|------|---------|------|
| **akshare** | 免费 | 另类数据、宏观指标 | 不能在策略循环里直接调用，必须离线 ETL 到本地数据库 |
| **tushare** | 需积分（基本免费） | A 股日频数据、基本面、财务数据 | 实盘首选，数据质量高 |
| **聚宽** | 回测免费，实盘按笔收费 | 快速验证策略，自带回测环境 | 对接 10+ 家券商 |
| **米筐** | 付费 | 分钟级数据，机构级质量 | 比聚宽贵，数据更准 |

**推荐：tushare + akshare 组合**，tushare 做主数据源，akshare 补充免费另类数据。

## 交易接口

### QMT vs PTrade

| | QMT | PTrade |
|--|-----|--------|
| 运行方式 | 本地运行 | 云端/券商服务器 |
| 延迟 | 1ms 以内 | 略高 |
| 适合频率 | 中高频 | 中低频 |
| 断网影响 | 策略停止 | 不影响 |
| Python 支持 | MiniQMT + XtQuant | 支持 |

**推荐 MiniQMT**（几百万资金，已有实盘经验），可以直接在 VSCode 里写 Python，通过 XtQuant 库下单，和 TradingView + Webhook 的方式最接近。

**开通门槛：**
- 最低 10 万（国金证券）
- 几百万资金随便选券商，华泰、东方财富、国泰君安都支持
- 线上申请，1-3 个工作日审核

## 回测工具

| 工具 | 适用场景 | 说明 |
|------|---------|------|
| vectorbt | ETF 轮动、多因子选股 | 向量化计算，速度快，代码简洁，首选 |
| Qlib（微软） | 多因子选股（因子积累后） | 内置 Alpha158/Point-in-Time/IC 检验，可接 RD-Agent |
| backtrader | 跳过 | 事件驱动，适合复杂订单逻辑，本项目场景用不上 |
| vnpy | 商品期货 CTA | 对接 CTP 实盘，自带回测模块，期货一体化方案 |

**路径：** vectorbt 起步 → Qlib（因子积累后迁移）→ RD-Agent（全自动因子挖掘）



```
数据获取（tushare/akshare）
    ↓ 盘后 ETL
本地数据库（ClickHouse 或 SQLite）
    ↓
Python 策略（VSCode + MiniQMT）
    ↓ XtQuant 下单接口
券商 QMT 柜台
    ↓
交易所撮合
```

**云服务器：** 需要，QMT 本地运行模式要求交易时间电脑常开，用云服务器更稳定，配置 i5 以上、16GB 内存够用。

## A股特殊限制

- 无公开交易所 API，个人只能通过券商客户端接口
- T+1 限制，涨跌停板，需在策略中特殊处理
- 程序化交易需要稳定运行环境（建议 Windows 云服务器，QMT 仅支持 Windows）

## 建议路径

1. 先在聚宽上用历史数据验证策略逻辑（免费，门槛低）
2. 开通支持 QMT 的券商账户（华泰或东方财富）
3. 本地/云服务器部署 MiniQMT + Python 环境
4. 模拟盘跑 1-2 周确认稳定后切实盘
