# 量化研究工作流规范

## 研究日志记录规范

每次完成有实质结论的研究，除了结论本身，还需要记录：

### 必须记录的内容

| 项目 | 说明 | 示例 |
|------|------|------|
| 数据范围 | 时间跨度、标的、数据源 | 2019-2026，BTC/ETH/SOL/DOGE，Binance 2h K线 |
| 测试次数 | 扫描了多少个参数组合 | 4N × 3阈值 × 3时间框架 = 36配置/标的 |
| 参数选择理由 | 为什么选这个参数而不是最优参数 | CI(10) 而非 CI(7)，因为跨标的更稳定 |
| 样本外情况 | 是否有保留数据未参与选参 | 无样本外验证，结论需谨慎 |
| 过拟合风险评估 | 测试次数多时需注明 | n_trials=144，最优结果有一定过拟合风险 |

### 研究日志追加格式（补充）

```
## YYYY-MM-DD｜[研究主题]

**结论：** [一句话核心结论]
**数据：** [时间范围、标的数量、总交易笔数]
**测试规模：** [参数组合数量，n_trials=N]
**参数选择理由：** [为什么选这个参数]
**过拟合风险：** [低/中/高，说明原因]
**样本外验证：** [有/无，说明]
**排除的方向：** [尝试了但没用的方法，说明为什么排除]
**细节：** [关键数据或发现，2-3 条]
**下一步：** [值得继续 / 放弃 / 待验证，说明原因]
```

## 脚本和结果持久化规范

### 问题背景

Claude context compact 后只有摘要，中间结果和脚本会丢失，导致研究无法继续或需要重跑。

### 规范

**脚本：**
- 分析脚本保存到 `crypto/analysis/`，不只放 `/tmp`
- 命名规范：`{主题}_{功能}.py`，如 `ci_filter_per_symbol.py`、`ci_filter_param_sweep.py`

**结果：**
- 重要的中间结果保存到文件（`/tmp/result_xxx.txt` 或项目目录）
- 最终结果写入研究日志

**研究结束检查清单：**
- [ ] 研究日志已更新（含测试次数、参数选择理由、排除的方向）
- [ ] memory 文件已更新（结论、roadmap）
- [ ] 分析脚本已保存到 `crypto/analysis/`
- [ ] CLAUDE.md 是否需要更新

**季度复盘检查清单（每季度末执行一次）：**
- [ ] 回顾过去 3 个月的研究日志，标记哪些结论已被新数据推翻
- [ ] 对标注"待验证"的结论，确认是否已有后续数据支撑或否定
- [ ] 更新 memory 文件中过时的结论

## 回测可复现性规范

### 参数记录

不要硬编码参数，脚本顶部统一声明：

```python
CI_N = 10
CI_TF = '2h'
CI_THRESH = 50
```

### 数据版本

每次研究注明：
- 数据来源（Binance/OKX）
- 时间范围（2019-01-01 至今）
- xlsx 文件名（含 hash，如 `strategy_ema_btc_OKX_BTCUSDT.P_2026-05-22_19c0f.xlsx`）

### 多重测试警告

测试组合数 > 10 时，在研究日志中注明：
- 总测试次数 n_trials
- 最优结果是否在样本外验证过
- 如未验证，结论标注"有过拟合风险，待实盘验证"

## 跨会话连续性规范

### compact 前

研究进行中时，定期把中间结果写到文件：

```python
# 每跑完一个重要步骤，保存结果
with open('/tmp/research_progress.txt', 'a') as f:
    f.write(f'[{step}] {result}\n')
```

### compact 后恢复

1. 读 `/tmp/research_progress.txt` 或已保存的结果文件
2. 读 `memory/MEMORY.md` 确认当前研究状态
3. 从上次中断点继续，不重复已完成的步骤

## CI 过滤研究结论（2026-05-27，参考案例）

作为本规范的第一个完整案例记录：

- **数据**：2019-2026，BTC/ETH/SOL/DOGE，v1/v2/v3_3h/v3_205m，共约 8000+ 笔交易
- **测试规模**：n_trials = 144（4N × 3阈值 × 3时间框架 × 4标的）
- **结论**：CI(10) 2H ≤50 有效，v1 提升 +57%
- **参数选择理由**：CI(10) 而非各标的最优参数（CI(7)/CI(14)），因为 N=10 跨标的最稳定；2H 而非 1H，因为 2H 在所有策略版本上均正向；阈值 50 而非 45/55，因为 50 在年度稳定性上最好
- **过拟合风险**：中等，n_trials=144，但规律（被过滤交易胜率 19-27%）在所有标的和年份上高度一致，降低了过拟合可能性
- **样本外验证**：无，所有年份均参与参数选择；2025 年略负（-2,732）可视为部分样本外信号
- **待验证**：Pine Script `f_ci(10)` vs `f_ci(10)[1]` 回测行为差异
