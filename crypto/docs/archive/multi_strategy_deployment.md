# 多策略并行部署：决策与注意事项

最后更新：2026-05-22

## 背景

原来每个标的只跑 V1（EMA）策略。本次分析基于 2026-05-18 回测数据，评估切换到每个标的同时跑 V1+V2+V3 的可行性。

---

## 切换决策

**结论：切换到每个标的同时跑 V1+V2+V3，分批切换。**

### 仓位方案

| 阶段 | V1 | V2 | V3 | 时间 |
|------|----|----|-----|------|
| 第一步（现在） | 70% | 15% | 15% | 观察 2-3 个月 |
| 第二步（视情况） | 40% | 30% | 30% | 等权方向 |

**为什么分批而不是直接等权：** 减少心理压力，不是因为数据不支持等权。6年回测显示等权组合全面优于纯 V1（CAGR 29.4% vs 21.3%，Sharpe 2.46 vs 1.68）。

### 评判标准（提前锁定）

- 评判周期：6 个月
- 干预线：实盘回撤超过历史最大回撤的 1.5 倍才考虑暂停
- 不因某个月 V1 跑赢而回调权重

## 手续费分析（独立跑 vs 合并信号）

每标的 10,000 USDT，手续费率万分之八（含滑点）。

| 标的 | V1 单独 | V2 单独 | 独立合计 | 合并跑 | 节省 |
|------|---------|---------|---------|--------|------|
| BTC | 2,293 | 1,768 | 4,062 | 3,924 | 138 |
| ETH | 2,324 | 2,467 | 4,791 | 4,490 | 301 |
| SOL | 2,935 | 2,530 | 5,465 | 5,130 | 335 |
| 合计 | | | 14,318 | 13,544 | **774** |

**结论：合并信号每年只节省 774 USDT，不值得做。**

原因：
- V3 与 V1/V2 信号时间完全不重叠（重合次数为 0），合并无意义
- V1/V2 虽有大量同时进场，但大部分同方向，冲突少，节省有限

---

## 资金费率影响

双向持仓时（V1 多 + V2 空同时持有），资金费率一边付一边收，净损失为 0（前提是仓位大小相等）。

OKX 永续合约 Hedge Mode 下，同方向仓位不会自动合并，两个 Bot 各自独立持仓。

---

## OKX Signal Bot 隔离问题（重要，待验证）

### 官方文档说明

- 信号策略强制 One-way Mode，同一标的只能持有一个方向
- EXIT 信号默认平掉 100% 净仓位

**风险：** Bot A 和 Bot B 对同一标的（如 BTC）同时做多，Bot A 发出 EXIT 信号，可能把 Bot B 的仓位也一起平掉。

### 文档来源

- https://www.okx.com/zh-hans/help/trading-signal-bot-faqs
- https://www.okx.com/help/signal-bot-alert-message-specifications

### 待验证

用小仓位（各 100 USDT）实测：两个 Bot 同标的同时做多，触发一个 EXIT，看另一个是否受影响。

**在确认隔离问题之前，不要把多个策略部署到同一标的。**

实测结果决定后续方案：
- 如果隔离正常 → 直接独立 Bot 运行
- 如果 EXIT 互相影响 → 考虑合并信号或 API 自管理

| 组合 | 净利润 % | 最大回撤 % | 夏普 |

**结论：SOL 最优方案为 v2+v3_205m 等权。** ema 与 v2 相关性 0.923，v2 各项指标更好，ema 可以放弃（或加入以小幅降低回撤）。
