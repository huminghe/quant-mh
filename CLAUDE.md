# quant-mh — CLAUDE.md

## 项目概述

量化交易研究与策略代码库，个人实盘项目，涵盖 A 股量化和加密货币策略两条线。

## 背景与现状

- 已有实盘经验：TradingView Pine Script 双向趋势策略 + Webhook → OKX/Binance（加密货币，约30个标的）
- 目标：将量化能力迁移到 A 股，研究数据获取、策略开发、回测、实盘执行全链路
- 资金量级：百万人民币级别

## 技术栈

### A 股
- **语言**：Python
- **数据源**：tushare（日线/财务/资金流）、akshare（免费补充数据）
- **交易接口**：待定（MiniQMT/XtQuant 或 XTP，见下方选型说明）
- **回测**：vectorbt（主力，多因子/ETF轮动）→ Qlib（因子积累后迁移）；期货用 vnpy 自带回测
- **部署**：本地 Mac 开发，实盘考虑云服务器（Linux 优先）

### 加密货币
- **策略**：双向趋势跟踪（做多+做空），Pine Script v6，约30个标的
- **执行**：TradingView Webhook → OKX/Binance 全自动
- **分析**：Python（`crypto/analysis/strategy_analysis.py`）
- **文档**：暂停机制（`crypto/docs/pause_mechanism.md`）、健康度监控（`crypto/docs/health_monitor.md`）

## A股交易限制（重要背景）

- 无公开交易所 API，个人只能通过券商客户端接口
- T+1 限制，涨跌停板，需在策略中特殊处理
- 高频策略受印花税（千1，仅卖出）严重压制，个人量化更适合中低频

## 数据源说明

| 数据源 | 类型 | 说明 |
|--------|------|------|
| tushare | 付费（积分制） | 日线、财务、资金流、龙虎榜等，质量高，主数据源 |
| akshare | 免费 | 爬虫聚合，覆盖广但稳定性一般，接口经常变动需及时升级 |
| MiniQMT | 实时 | 通过本地客户端获取 tick/分钟线，仅实盘用 |
| 聚宽 | 云端平台 | 自带回测环境，适合早期验证策略逻辑，代码在云端有泄露风险 |
| 米筐 | 付费云端 | 分钟级数据质量高，价格不透明需联系销售，同样有泄露风险 |

## 交易接口选型

### 股票/ETF/可转债接口

| 接口 | 系统要求 | 部署方式 | 自定义程度 | 策略泄露风险 |
|------|---------|---------|-----------|------------|
| MiniQMT + XtQuant | 仅 Windows | 本地/Windows云服务器 | 完全自由 | 低 |
| PTrade（恒生） | 无需关心 | 券商云端，无需维护 | 受限（框架约束、库受限） | 有 |
| XTP（中泰证券） | Windows + Linux | 本地/任意云服务器 | 完全自由 | 低 |

**结论：**
- Linux 云服务器优先 → 中泰证券 XTP（官方 Python API，原生支持 Linux）
- 不介意 Windows → MiniQMT，社区更活跃，资料更多
- 不想维护运行环境 → PTrade，但代码在券商服务器，自定义受限

**XtQuant 工作方式：**
Python 策略脚本通过本地 IPC 与 MiniQMT 客户端通信下单，类似之前 TradingView Webhook → 交易所的模式，但脚本和客户端必须在同一台机器上。

### 期货接口（商品期货 / 股指期货）

期货走独立通道，与股票账户完全分离：

| 接口 | 说明 |
|------|------|
| CTP（穿透式监管） | 国内期货自动交易行业标准，几乎所有期货公司支持 |
| vnpy | 基于 CTP 的开源量化框架，Python，社区活跃，推荐使用 |
| openctp | CTP 仿真环境，免费，Mac 可用，用于开发测试 |

**开户流程：** 期货公司开户 → 联系客户经理申请 CTP/API 权限（1~3 个工作日）→ 获得服务器地址和经纪商代码 → vnpy 接入

**注意：** Mac 无法跑实盘 CTP，实盘需部署到 Linux 云服务器或 Windows 机器。

- 纯股票高频：印花税压制，个人基本跑不赢摩擦成本

## A 股趋势策略失效的根本原因

趋势策略在 A 股个股上失效，不是参数问题，是市场结构问题：

- **T+1**：买入当天无法卖出，止损成本远高于回测假设
- **无法做空**：趋势策略少了一半机会，下跌趋势完全无法参与
- **散户主导**（60% 成交量）：散户追涨杀跌制造短期过度反应（1~20 个交易日）+ 中期均值回归（1~6 个月），而非趋势延续
- **政策市**：国家队托底随时截断下跌趋势
- **涨跌停板**：价格发现延迟，趋势信号滞后且无法及时执行

**A 股期货没有上述限制**，商品期货 CTA 是趋势策略在 A 股生态下的正确载体。

## 策略方向

基于调研结论，确定以下策略组合方向：

| 策略 | 仓位 | 接口 | 优先级 |
|------|------|------|--------|
| ETF 趋势轮动 | 30% | MiniQMT | 第一步，逻辑简单，验证框架 |
| 指数增强（多因子选股） | 30% | MiniQMT | 第二步，聚焦反转+质量因子 |
| 商品期货 CTA | 30% | CTP + vnpy | 可复用加密货币趋势策略经验 |
| 可转债双低 | 10% | MiniQMT | 降低整体波动，需过滤低评级 |

**策略选择依据：**
- 趋势策略经验迁移：加密货币趋势策略 → 商品期货 CTA（两者市场结构相似）
- A 股个股不适合趋势策略，ETF 轮动是跨资产趋势策略的替代
- 一两百万资金不适合市场中性策略（股指期货对冲粒度粗 + 基差成本高），指数增强更合适
- CTA 与股票策略相关性低，熊市时天然对冲

**多因子策略注意事项：**
- 有效因子：短期反转（A 股散户过度反应）、质量（ROE 稳定性）、低波动、盈利质量（经营现金流/净利润匹配度）
- 已失效因子：小市值（2024 年 2 月踩踏证伪）、简单低估值、价格动量（A 股整体是反转市场，正向动量无效）
- 残差动量（剥离市场/行业 beta 后）是正向的，可作为独立因子使用
- 选股池限定沪深 300 / 中证 500，避免小盘股退市风险
- 月度调仓，年换手率控制在 200~400%
- 财务数据季度披露，需用 point-in-time 数据避免前视偏差，安全缓冲期 45-60 天
- 月度调仓的驱动力：价量因子（每月变化）+ 财务因子（季度冻结，提供质量筛选）

**ETF 轮动注意事项：**
- 本质是系统化资产配置，不是高频量化
- 无印花税，成本最低，适合作为验证框架的第一个策略
- 月度调仓，动量信号窗口约 20-30 日
- 标的池 13-15 只，覆盖宽基、行业、跨境、避险

**回测工具选型结论：**
- vectorbt：多因子/ETF 轮动主力回测工具，向量化快速，代码简洁
- Qlib：因子积累后迁移，内置 Alpha158、Point-in-Time、IC 检验，后续可接 RD-Agent
- backtrader：跳过，场景与本项目不匹配
- vnpy：期货 CTA 回测 + 实盘，一体化

**AI 因子挖掘：**
- 半自动流程（现在可用）：人工找论文/帖子 → AI 解读生成 vectorbt 代码 → 自己跑回测验证
- 论文来源：SSRN q-fin、arXiv q-fin、聚宽社区、BigQuant、"因子动物园"公众号、券商金工研报
- 全自动流程（后期）：Qlib + RD-Agent，因子积累到一定数量后再考虑
- RD-Agent 默认基于 Alpha158，需要自定义输入方向才能避免同质化

**风险控制规则化：**
- 策略净值从高点回撤超过 15% 暂停交易
- 连续 3 个月跑输沪深 300 超过 5% 触发审查
- 因子 IC 连续 2 个月为负触发审查
- 沪深 300 跌破 200 日均线时降至半仓（可选）
